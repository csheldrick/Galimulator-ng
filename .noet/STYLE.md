# STYLE

## Desired qualities
- Observer-first, not player-progression-first.
- Emergent history over scripted story.
- Strange but legible galactic events.
- Systems should produce memorable narratives from simple rules.
- Names, dynasties, religions, crises, wars, rulers, monsters, and collapses should feel like a living chronicle.
- The best feature is one the user can notice by watching the galaxy unfold.
- Mechanics should create visible consequences: event log entries, altered empire behavior, changed relationships, map changes, or history report details.
- Prefer one tight causal loop over a pile of loosely related features.

## Undesired qualities
- Generic 4X mechanics.
- Feature checklist thinking.
- UI panels that expose numbers without improving the fantasy.
- Mechanics that only exist internally and never surface as events, behavior, or map changes.
- Adding more stats without creating more lived-history feel.
- Turning the project into Stellaris/Civilization instead of preserving the Galimulator observer vibe.
- Big architecture rewrites for a small simulation-feel improvement.

## Good examples
- A dynasty remembers that another empire executed its ruler, raising future tension and producing revenge-flavored event text.
- A holy world massacre becomes a named historical atrocity that later fuels crusades or rebellions.
- An empire’s golden age is remembered in its history report and affects future legitimacy or nostalgia events.
- A defeated ruler’s heir later returns as a rebel claimant with an event trail that makes the rebellion feel earned.
- A plague, monster attack, or ancient awakening leaves a remembered scar that changes local culture, religion, stability, or diplomacy.

## Failure examples
- “Add espionage, resources, diplomacy, ship upgrades, and quests” without one coherent mechanic.
- Add hidden `memoryScore` fields that never affect behavior, events, inspectors, or exports.
- Add a large UI history panel before the simulation produces meaningful remembered history.
- Implement generic tech-tree or economy mechanics that do not strengthen the observer-driven chronicle feel.
- Add random lore text that is not connected to actual simulation state.
